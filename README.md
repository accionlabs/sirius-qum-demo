# Quantitative User Metrics &mdash; Hubexo Lead Manager Demo

A static GitHub Pages site that wraps the **Quantitative User Metrics (QUM)**
report generated during Accion Labs' Project Sirius assessment of Hubexo's
Lead Manager application.

## What's here

```
qum-demo-site/
├── index.html                # Landing page with QUM context + embedded report
├── assets/                   # Styles, images, scripts for the landing page
└── report/                   # The original QUM build output (unmodified)
    ├── quantitative-user-metrics/   # Vue SPA entry point
    ├── qumReportSummaryPathDetails.js
    ├── trendsChartSummary.js
    └── 20260415_123025/             # Dated run: data, captures, accessibility reports
```

## How the report is wired

The report is a Vue SPA generated by [Nexial](https://nexiality.github.io/documentation/),
Accion Labs' open-source test automation platform. It loads run metadata from
`report/qumReportSummaryPathDetails.js` which points at the dated execution
folder. The landing page embeds the SPA in an iframe and also links out to it.

## Running locally

No build step. Just serve the folder:

```bash
python3 -m http.server 8000
# open http://localhost:8000
```

## Deployment

This repo is published to GitHub Pages from the `main` branch root. The
`.nojekyll` file is required so that Pages serves the `_`-free SPA paths
correctly and does not strip any folders.
